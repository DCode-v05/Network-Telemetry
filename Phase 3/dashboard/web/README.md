# Phase 3 Dashboard — "Ensemble Command" (React + Vite + ECharts)

An interactive, single-file dashboard for the Phase 3 two-layer ensemble
benchmark. Replaces the old Plotly HTML report with a custom-designed React app.

- **Stack:** React 18, [Vite](https://vitejs.dev/), [Apache ECharts](https://echarts.apache.org/) (`echarts-for-react`)
- **Output:** `npm run build` emits **one self-contained `dist/index.html`** (JS + CSS inlined via `vite-plugin-singlefile`) — open it directly, no server needed.
- **Data:** baked in at build time from `src/data.json`, produced by `../export_data.py` (which reuses `evaluation/phase3_metrics.py`).

## Workflow

```bash
# 1. (Re)generate the data snapshot from the latest results CSVs.
#    Use the Python env that has pandas (../../Phase 2/env).
python ../export_data.py            # writes src/data.json

# 2. Install deps (first time only).
npm install

# 3a. Develop with hot reload.
npm run dev

# 3b. Build the self-contained dashboard.
npm run build                       # -> dist/index.html
```

The export also pulls in `../../Phase 2/results/csv/aggregated_results.csv`
when present, to drive the Phase 2 → Phase 3 comparison figure.

## What it shows

- Animated KPI strip (mean gate FP reduction, ensemble recall, ensemble FPR, runs)
- **Two-layer architecture diagram** (Spike ∧ / Sustained ∨ → OR fusion, with gates)
- **Confirmation-gate effect** — false positives eliminated vs true positives kept
- **Ensemble vs best single detector** (TPR / FPR / F1 toggle)
- **Phase 2 → Phase 3** lineage comparison
- 14-detector Performance Matrix, TPR/FPR (group-filtered), latency, radar
- Full detector leaderboard ranked by false-alarm rate
